# Credit Card Fraud Detection — Production ML Project

A refactored, production-ready version of the fraud detection notebook.  
Used as the running project across the **DevOps Foundations for ML** course series.

---

## Project Structure

```
fraud-detection/
├── data/                        ← gitignored — download from Kaggle
│   └── creditcard.csv
├── models/                      ← gitignored — generated by training
│   ├── best_model.pkl
│   └── model_meta.json
├── notebooks/
│   └── exploration.ipynb        ← original EDA notebook (read-only reference)
├── reports/                     ← generated plots and metrics
├── src/
│   ├── config.py                ← all constants in one place
│   ├── preprocess.py            ← load, validate, split data
│   ├── train.py                 ← train 4 models, select best by AUC
│   ├── evaluate.py              ← metrics, confusion matrix, ROC, PR curves
│   └── predict.py               ← inference — single transaction or batch
├── tests/
│   ├── test_preprocess.py
│   └── test_predict.py
├── run_pipeline.py              ← run everything end-to-end
├── requirements.txt
└── .gitignore
```

## Setup

```bash
# 1. Clone the repo
git clone <your-repo-url>
cd fraud-detection

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download dataset from Kaggle
#    https://www.kaggle.com/mlg-ulb/creditcardfraud
#    Place creditcard.csv in data/
```

---

## Usage

### Run the full pipeline (preprocess → train → evaluate → plots)
```bash
python run_pipeline.py
```

### Run with a custom fraud threshold
```bash
python run_pipeline.py --threshold 0.3
```

### Run individual steps
```bash
python src/preprocess.py     # check data loading and splits
python src/train.py          # train all models, save best
python src/evaluate.py       # evaluate saved model on test set
python src/predict.py        # demo: score a single transaction
```

### Run tests
```bash
pytest tests/ -v
```

## Dataset

**Credit Card Fraud Detection**  
284,807 transactions · 492 frauds (0.172%) · 30 features (PCA-transformed)  
Source: https://www.kaggle.com/mlg-ulb/creditcardfraud

> The `data/` directory is gitignored. Use DVC or cloud storage for data versioning.
